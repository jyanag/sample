たとえるなら「料理」の世界

4つの関係は、料理にたとえるとイメージしやすくなります。全員が「JavaScript という同じ言語」の上に成り立っている、という点がポイントです。

🔪

言語 = 調理の基本技術

JavaScript

「切る・焼く・煮る」のような基本技術そのもの。これがないと何も作れない、すべての土台です。

🍳

ライブラリ = 便利な調理器具

React

画面(UI)を効率よく組み立てるための便利な道具セット。使うかどうかは自由ですが、あると圧倒的にラクです。

🏠

フレームワーク = システムキッチン

Next.js

React という調理器具に、コンロ・収納・動線まで最初から揃えた「型」。決められた作法に従えば、本格的なWebアプリが早く作れます。

🏭

実行環境 = 家の外のキッチン

Node.js

本来ブラウザの中でしか動けなかった JavaScript を、サーバー(PC)側でも動かせるようにする仕組みです。

ひと目でわかる比較表

「種類」の列に注目してください。React・Next.js・Node.js は言語ではなく、JavaScript を活用するための道具や環境です。

名前種類主な役割動く場所
JavaScript プログラミング言語 すべての土台。文法・変数・関数など ブラウザ / サーバー
React ライブラリ UI(画面の部品)を効率よく作る 主にブラウザ
Next.js フレームワーク React を使ったWebアプリを丸ごと作る(ルーティング・SSRなど) ブラウザ + サーバー
Node.js 実行環境 ブラウザの外(サーバー)で JavaScript を動かす サーバー / PC

よくある勘違い

初心者がつまずきやすいポイントを先回りで解消しておきましょう。

?

「React 言語」「Next.js 言語」という言語がある?

ありません。React も Next.js も、書いているコードはあくまで JavaScript(+ TypeScript)です。だから図でも、大きな JavaScript の円の「中」に描かれています。

?

React と Next.js はライバル関係?

むしろ親子のような関係です。Next.js は React の上に作られたフレームワークなので、Next.js を使うときは自動的に React も使っています(図の点線がその関係です)。

?

Node.js はブラウザで動く?

逆です。Node.js は「ブラウザの外」で JavaScript を動かすためのもの。Next.js のサーバー側の処理も、この Node.js の上で動いています。

まずは土台の JavaScript から。円の内側の道具たちは、その後で必ず理解できるようになります 💪
